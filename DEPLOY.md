# Деплой (24/7, доступ с телефона)

Цель: бот и панель крутятся на сервере круглосуточно, а ты заходишь с телефона по
ссылке, добавляешь чаты и расписания. Сообщения уходят по таймеру, даже когда
твой компьютер выключен.

> **Важно про безопасность.** Панель даёт полный доступ к Telegram-аккаунтам.
> Вход — по **регистрации с одобрением**: люди регистрируются (логин + пароль),
> но войти могут только после того, как ты (первый зарегистрированный = админ)
> одобришь их в разделе «Пользователи». Заходить нужно по **HTTPS** (Caddy
> получает сертификат автоматически для домена из `PANEL_DOMAIN`).

Дальше два варианта хоста. **Из России работает «Российский VPS»** (Oracle/AWS/GCP
не регистрируют пользователей из РФ). Шаги с Docker (4–7) у обоих общие.

---

## Вариант A: российский VPS (рекомендуется для РФ)

Дешёвый VPS ~100–150 ₽/мес, оплата картой **Мир**. Провайдеры: **aeza**, **vdsina**,
**timeweb.cloud**, **ruvds**, **selectel**.

1. Зарегистрируйся у провайдера, оплати картой Мир.
2. Создай сервер: **Ubuntu 22.04 (или 24.04)**, минимальный тариф
   (**1 vCPU, 1 GB RAM** — этого хватает с запасом). Локация — Россия/ближе к тебе.
3. Получишь **публичный IP** и **root-пароль** (или загрузишь свой SSH-ключ) — обычно
   приходят на почту/в панели сразу.
4. Подключись по SSH:
   ```bash
   ssh root@ТВОЙ_IP        # пароль из письма провайдера
   ```
5. **Порты open by default** — у большинства VPS firewall выключен, отдельно открывать
   80/443 не нужно (если стоит `ufw` — `sudo ufw allow 80,443/tcp`).
6. Дальше сразу переходи к **разделу 4 (Установить Docker)** ниже.

> На VPS ты обычно `root`, поэтому в командах ниже `sudo` можно опускать, а имя
> пользователя `ubuntu` заменить на `root`.

## Вариант B: Oracle Cloud Always Free (НЕ для России)

## 1. Создать аккаунт Oracle Cloud

1. Зайди на https://www.oracle.com/cloud/free/ → **Start for free**.
2. Регистрация: email, страна, телефон, **банковская карта** (для верификации;
   с Always Free деньги не списываются).
3. Выбери домашний регион поближе к тебе.

## 2. Создать бесплатную виртуальную машину

1. Меню → **Compute → Instances → Create instance**.
2. **Image**: Ubuntu 22.04 (или 24.04).
3. **Shape**: жми *Change shape* → раздел **Ampere (Arm)** → `VM.Standard.A1.Flex`,
   поставь **1 OCPU / 6 GB RAM** (всё в рамках Always Free). Если Ampere «нет в
   наличии» — пробуй другой момент времени или регион, либо возьми
   `VM.Standard.E2.1.Micro` (x86, тоже Always Free, послабее).
4. **SSH keys**: *Generate a key pair* → скачай приватный ключ (понадобится для входа).
5. **Create**. Через минуту увидишь **Public IP address** — запиши его.

## 3. Открыть порты (два места!)

Oracle по умолчанию закрывает всё. Нужно открыть.

**а) В облаке (Security List / NSG):**
1. На странице инстанса → раздел **Virtual cloud network** → подсеть → **Security List**.
2. **Add Ingress Rules**:
   - Source `0.0.0.0/0`, IP Protocol TCP, Destination port **80** и **443**
     (для HTTPS через Caddy).
   - (Опционально для быстрого теста) порт **8000**.

**б) На самой машине (iptables в Ubuntu).** Подключись по SSH:
```bash
ssh -i /путь/к/ключу ubuntu@ТВОЙ_PUBLIC_IP
```
И открой порты:
```bash
sudo iptables -I INPUT 6 -m state --state NEW -p tcp --dport 80 -j ACCEPT
sudo iptables -I INPUT 6 -m state --state NEW -p tcp --dport 443 -j ACCEPT
sudo netfilter-persistent save
```

## 4. Установить Docker

На машине (по SSH):
```bash
curl -fsSL https://get.docker.com | sudo sh
sudo usermod -aG docker ubuntu
newgrp docker            # или выйди и зайди по SSH заново
```

## 5. Домен (DuckDNS)

Чтобы заходить с телефона по HTTPS, нужен домен, указывающий на сервер:

1. Зарегистрируйся на https://www.duckdns.org (вход через Google/GitHub).
2. Создай поддомен, например `mybot`, и в поле *current ip* впиши **Public IP
   сервера**, нажми *update ip*. Получишь адрес `mybot.duckdns.org`.

## 6. Загрузить проект и запустить

```bash
# поставить git, если нет
sudo apt-get update && sudo apt-get install -y git

# забрать код
git clone ВАШ_РЕПОЗИТОРИЙ yoy
cd yoy

# создать .env: домен и часовой пояс
cp .env.example .env
nano .env        # PANEL_DOMAIN=mybot.duckdns.org, TZ=Europe/Moscow

# собрать и запустить (в фоне, с автоперезапуском)
docker compose up -d --build
```

Caddy (входит в `docker-compose.yml`) сам получит HTTPS-сертификат для
`PANEL_DOMAIN`. Проверь логи:
```bash
docker compose logs caddy --tail 20   # ищем "certificate obtained successfully"
```

Открой на телефоне **https://mybot.duckdns.org** — откроется панель.

## 7. Регистрация, одобрение и вход в Telegram

1. На сайте **зарегистрируйся** (логин + пароль). **Первый зарегистрированный
   становится администратором** — это ты.
2. Другие люди тоже регистрируются, но попадают в статус «ждёт одобрения».
   Зайди как админ → кнопка **«Пользователи»** → **«Одобрить»** нужных (или
   «Заблокировать»/«Удалить»). Войти и пользоваться могут только одобренные.
3. Каждый пользователь видит **только свои** Telegram-аккаунты и расписания.
4. Добавление Telegram-аккаунта: «＋ Добавить аккаунт» → `api_id`/`api_hash`
   (с https://my.telegram.org) → телефон → код → пароль 2FA. Дальше — поиск
   чатов и расписания.

Профили, расписания, пользователи и ключ сессий хранятся в томе `./profiles`
на сервере и переживают перезапуски и пересборку контейнера.

## Обновление кода на сервере

Файлы `docker-compose.yml`/`Caddyfile` на сервере могли создаваться вручную —
чтобы `git pull` не конфликтовал, обновляйся так:

```bash
cd ~/yoy
rm -f Caddyfile docker-compose.yml     # уберём ручные версии (они есть в репозитории)
git fetch origin && git reset --hard origin/main
# если в .env ещё нет PANEL_DOMAIN — добавь:
nano .env
docker compose up -d --build
```

`.env` и папка `profiles/` (аккаунты, пользователи) при этом не трогаются.

## Частые вопросы

- **Сообщения уходят не в то время** → проверь `TZ` в `.env` и пересоздай контейнер
  (`docker compose up -d`).
- **Сайт не открывается** → домен не указывает на сервер (шаг 5), либо порт не открыт
  (шаг 3, для Oracle — оба пункта).
- **Забыл пароль администратора** → останови (`docker compose down`), удали
  `profiles/users.json` (внимание: сотрёт всех пользователей, профили Telegram
  останутся и привяжутся к новому первому админу), запусти заново и зарегистрируйся.
- **Аккаунт Telegram заблокировали** → userbot нельзя использовать для спама/массовых
  рассылок, за это банят. Это ограничение Telegram, а не хостинга.
